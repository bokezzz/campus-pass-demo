# 后台角色菜单与友好权限提示

## 目标

后台根据当前管理员角色显示可访问的模块；无权限访问模块时显示友好提示页，不再直接返回 HTTP 403 错误页。

## 角色菜单

| 角色 | 可见模块 |
| --- | --- |
| 系统管理员 | 首页、部门管理、管理员管理、社会公众预约、公务预约、审计日志 |
| 学校管理员 | 首页、部门管理、管理员管理、社会公众预约、公务预约、审计日志 |
| 部门管理员 | 首页、部门管理、公务预约 |
| 审计管理员 | 首页、审计日志 |

`publicReservationPermission` 为真时，额外显示社会公众预约入口。

## 访问控制

- `AuthFilter` 保留登录检查。
- 新增角色权限过滤器，拦截后台模块路径；无权限时转发到统一提示页。
- `AdminReservationsServlet` 保留对社会公众预约的数据权限检查，作为业务层防线。
- 部门管理员查询公务预约时，DAO 继续限制为本部门数据。

## 页面

- `nav.jsp` 根据当前管理员角色显示不同导航项。
- 新增 `forbidden.jsp`，显示当前模块无权限、返回后台首页按钮。

## 验证

- 为角色路径判断编写单元测试。
- 分别使用四个默认账号验证菜单和页面访问结果。
